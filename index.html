Giáo Trình Cơ Bản Về Git và GitHub: Từ A đến Z

Chào mừng các bạn đến với khóa học Git và GitHub! Trong giáo trình này, chúng ta sẽ cùng nhau khám phá Git - hệ thống quản lý phiên bản mạnh mẽ, và GitHub - nền tảng lưu trữ mã nguồn lớn nhất thế giới. Dù bạn là người mới bắt đầu hay đã có kinh nghiệm, khóa học này sẽ cung cấp cho bạn kiến thức nền tảng vững chắc để làm việc hiệu quả hơn trong các dự án cá nhân và nhóm.

1. Git & GitHub: Phân Biệt và Liên Kết

Trước tiên, chúng ta cần làm rõ hai khái niệm này. Đây là điều mà nhiều người mới học thường nhầm lẫn.

Git là gì?

Git là một hệ thống quản lý phiên bản phân tán (Distributed Version Control System - DVCS). Hãy hình dung nó như một cỗ máy thời gian dành cho mã nguồn của bạn. Nó cho phép bạn theo dõi mọi thay đổi, lưu lại các "bản chụp" (snapshot) của dự án theo từng mốc thời gian, và dễ dàng quay lại các phiên bản trước đó khi cần. Git được phát triển bởi Linus Torvalds (người tạo ra Linux) để quản lý mã nguồn của hệ điều hành Linux.

GitHub là gì?

GitHub là một nền tảng lưu trữ mã nguồn trực tuyến sử dụng Git. Nếu Git là cỗ máy thời gian trên máy tính của bạn, thì GitHub là "ngân hàng" lưu trữ an toàn các bản sao của cỗ máy đó trên internet. Nó cho phép nhiều người cùng làm việc trên một dự án, chia sẻ mã nguồn, và theo dõi tiến độ một cách hiệu quả.

Tóm lại:

  • Git là công cụ để quản lý mã nguồn trên máy tính cá nhân của bạn.
  • GitHub là dịch vụ để chia sẻ và cộng tác các dự án Git trên mạng.

2. Repository: Kho Chứa Mã Nguồn

Repository (hay kho chứa) là nơi Git lưu trữ toàn bộ lịch sử thay đổi của dự án. Nó chứa tất cả các tệp (file), thư mục và các thông tin meta của dự án.

Có hai loại repository chính:

  • Local Repository: Đây là kho chứa được tạo trên máy tính của bạn. Mọi thao tác như thêm file, sửa code, hay lưu lại lịch sử (commit) đều diễn ra trên local repository này.
  • Remote Repository: Đây là kho chứa được đặt trên một server từ xa (ví dụ như GitHub). Nó dùng để chia sẻ và đồng bộ mã nguồn giữa nhiều người trong một dự án.

3. Cài Đặt Git và Thiết Lập Ban Đầu

Để bắt đầu, bạn cần cài đặt Git trên máy tính của mình.

Bước 1: Cài đặt Git

  • Trên Windows: Truy cập http://git-scm.com/download/win, tải file cài đặt .exe và chạy.
  • Trên macOS: Bạn không cần phải dowload, trên máy đã có sẵn, nhiệm vụ tiếp theo của bạn là thiết lập thông tin cá nhân ban đầu.

Bước 2: Thiết lập thông tin cá nhân

Sau khi cài đặt, bạn cần khai báo tên và email. Điều này rất quan trọng vì nó sẽ được gắn vào mỗi "commit" bạn tạo ra (mục đích của nó cho biết rằng ai vừa mới cập nhật project). Việc này bạn chỉ cần làm 1 lần và hệ thống sẽ tự lưu lại.

Mở Terminal (trên macOS/Linux) hoặc Git Bash (trên Windows) và nhập các lệnh sau:

git config --global user.name "Tên Của Bạn" 
git config --global user.email "email_của_bạn@example.com"

Để kiểm tra lại các thiết lập, bạn có thể dùng lệnh:

git config --list

Điều này sẽ hiển thị các thông tin bạn vừa thiết lập, bao gồm tên và email. Bạn đã hoàn thành bước đầu tiên là cài đặt ban đầu, ở phần tiếp theo chúng ta sẽ tìm hiểu các câu lệnh cơ bản.

4. Bắt Đầu Với Các Lệnh Git Cơ Bản

1. Tạo một Repository

a) Tạo một Local Repository mới

Sử dụng câu lệnh init để khởi tạo 1 local repository mới:

git init

Sau câu lệnh init bạn vừa khời tạo 1 local repository, việc của bạn là conect nó với 1 remote repository trên GitHub, nếu bạn đã có sẵn thì hãy copy link và điền vào. Nếu chưa hãy truy cập GitHub.com -> Create a new repository -> Repository name -> Crate repository. Sau đó bạn sẽ có 1 link xuất hiện, hãy copy nó để conect với local repository của bạn thông quan câu lệnh phía dưới.

git remote add origin [URL_của_GitHub_Repo]

Sau câu lệnh này, local repository của bạn và remote repository đã được kết nối với nhau, khi ta thực hiện các câu lệnh thì có thể tương tác giữa các ripository.

Git Workflow: Quy trình làm việc hiệu quả

Đây là quy trình làm việc cốt lõi của Git. Chúng ta sẽ làm quen với các thành phần:

  • Working Directory: Là nơi bạn làm việc trực tiếp với các tệp tin của dự án. Đây là thư mục trên máy tính của bạn chứa tất cả các tệp và thư mục mà bạn đang chỉnh sửa, thêm, hoặc xóa. Bất kỳ thay đổi nào bạn thực hiện (ví dụ: viết code, sửa lỗi) đều nằm ở đây. Ban đầu, các thay đổi này chưa được Git theo dõi.
  • Staging Area: Staging Area, hay còn gọi là Index, là một khu vực trung gian. Khi bạn đã hoàn thành một phần công việc và muốn chuẩn bị lưu lại (commit), bạn sẽ dùng lệnh git add để "đưa" các tệp đã thay đổi vào khu vực này. Staging Area giống như một "trạm trung gian giữa local và remote" nơi bạn chọn lọc và sắp xếp các thay đổi để chuẩn bị push lên remote repository để lưu lại những chỉnh sửa gần đây với project. Điều này cho phép bạn nhóm các thay đổi có liên quan lại với nhau thành một commit duy nhất.
  • Local Repository: Là nơi Git lưu trữ tất cả lịch sử commit của dự án trên máy tính của bạn. Khi bạn thực hiện lệnh git commit, các thay đổi từ Staging Area sẽ được lưu lại vĩnh viễn vào Local Repository dưới dạng một commit (bản ghi) mới. Mỗi commit này có một mã định danh duy nhất (hash) và bao gồm thông điệp mô tả những thay đổi đã thực hiện. Đây là kho lưu trữ đầy đủ và an toàn nhất của dự án trên máy của bạn.

Lệnh git add

# Thêm một file cụ thể
$ git add ten_file_cu_the.txt

# Thêm tất cả các thay đổi
$ git add .

Lệnh git add dùng để thêm các thay đổi từ vùng làm việc (Working Directory) vào khu vực trung gian (Staging Area). Điều này giống như việc bạn chọn ra những tệp nào muốn lưu lại trong lần commit tiếp theo.

Lệnh git commit

$ git commit -m "Commit message"

Thông điệp (commit message): Một đoạn văn bản mô tả ngắn gọn và súc tích về những thay đổi đã được thực hiện trong commit này. Thông thường bạn sẽ mô tả những thay đổi với file đã add ở câu lệnh trước đó, chẳng hạn như "Create a new index.html file" hay bạn vừa sửa function A thì commit "Change function A"

Ở đây mình vừa add 1 file index.html và lưa commit rằng "Create a new index.html". Git phản hồi rằng đã thêm 1 file và thay đổi 10 dòng trong đó. Lúc này file và commit vẫn đang ở trong stage area, bạn và đồng nghiệp vẫn chưa thể thấy nó trên GitHub, việc tiếp theo của bạn và Push nó lên nhờ câu lệnh Push ở phía sau.

Lệnh git push

$ git push origin [tên_nhánh]

Lệnh git status

$ git status

Lệnh git pull và git fetch

$ git pull origin [tên_nhánh]

Các Lệnh Hữu Ích Khác

  • git diff: Xem sự khác biệt chi tiết.
  • git log: Xem lịch sử các commit.
  • git reset: Hoàn tác các commit.
